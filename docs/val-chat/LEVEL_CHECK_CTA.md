# Проверка уровня перед переходом на рекомендованный урок

**Дата реализации:** 15 декабря 2024  
**Статус:** ✅ Реализовано  
**Автор:** AI Code Agent

## Описание

Добавлена проверка текущего уровня пользователя перед переходом на рекомендованный урок из отчёта Валли.

## Проблема

При завершении валидации идеи, Валли рекомендует пользователю пройти определённый урок (level). Однако система не проверяла, достиг ли пользователь этого уровня, что могло привести к:
- Попытке открыть недоступный урок
- Плохому UX (пользователь попадает на экран с ошибкой)
- Путанице в навигации

## Решение

### Логика проверки

```dart
final currentLevel = await ref.read(currentLevelNumberProvider.future);

if (currentLevel < recommendedLessonLevel) {
  // Показываем SnackBar с сообщением
  ScaffoldMessenger.of(context).showSnackBar(
    const SnackBar(
      content: Text('Вы ещё не достигли этого уровня. Сначала пройдите предыдущие уровни.'),
      duration: Duration(seconds: 4),
    ),
  );
  return; // НЕ навигируем
}

// Уровень доступен → переходим
context.push('/levels/$levelNumber');
```

### Fail-safe поведение

В случае ошибки при загрузке уровня пользователя (например, проблемы с сетью), система всё равно выполнит переход на урок. Это гарантирует, что функционал не сломается полностью при технических проблемах:

```dart
try {
  // проверка уровня
} catch (e) {
  debugPrint('Failed to check user level: $e');
  context.push('/levels/$levelNumber'); // Всё равно переходим
}
```

## Риски и ограничения

### Риски

1. **Низкая вероятность:** Если `currentLevelNumberProvider` вернёт некорректное значение (0 или null), может сработать блок catch и пользователь всё равно перейдёт на недоступный урок
   - **Митигация:** провайдер нормализует значение и имеет breadcrumb для Sentry

2. **Низкая вероятность:** Race condition, если уровень пользователя изменится между загрузкой и проверкой
   - **Митигация:** окно очень маленькое (доли секунды), реальная вероятность близка к 0

### Ограничения

1. **Нет тестового покрытия:** Тесты заскипаны, требуется полное мокирование для реализации
2. **Fail-safe поведение:** При ошибке чтения уровня переход всё равно произойдёт
   - Это осознанный выбор: лучше дать пользователю попробовать, чем полностью блокировать функционал

## Дальнейшие улучшения (опционально)

1. **Добавить прелоадер:** Показывать индикатор загрузки при проверке уровня (если проверка занимает > 100ms)

4. **Метрики:** Логировать в Sentry/Analytics, как часто срабатывает блокировка перехода
