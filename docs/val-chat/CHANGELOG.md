Все изменения функционала Валли (AI-валидатор бизнес-идей).

## [Unreleased]

### 15 декабря 2024

#### ✅ Добавлена двухпроходная валидация качества ответов

**Описание:**  
Валли теперь не засчитывает нерелевантные или слишком общие ответы. Система проверяет каждый ответ пользователя на соответствие минимальным критериям перед переходом к следующему вопросу.

**Проблема (до):**
- Валли задавал фиксированно 7 вопросов по счётчику
- Если писать нерелевантную чушь — засчитывал как ответ и двигался дальше
- Качество финального скоринга страдало из-за неполных ответов

**Решение (после):**
- **Двухпроходная проверка** на backend (Edge Function):
  1. **Generator**: Валли генерирует ответ (как раньше)
  2. **Validator**: Отдельный вызов модели проверяет, достаточен ли ответ пользователя
- **Step gating**: Шаг (`current_step`) увеличивается **только** если `is_sufficient=true`
- **Мягкое добивание**: При недостаточном ответе Валли возвращает:
  - Короткий фидбек ("ответ слишком общий")
  - 2-3 уточняющих пункта
  - Мини-шаблон ответа
  - Перефразированный вопрос

**Минимальные критерии по шагам:**

1. **Суть идеи**: что создаётся + результат для пользователя + краткое описание
2. **Проблема**: кто страдает + как часто + почему больно (последствия)
3. **ЦА**: кто пользователь/покупатель + сегмент + где найти
4. **Валидация**: откуда знание + что проверял + сигналы
5. **Конкуренты**: как решают сейчас + почему плохо + альтернативы
6. **Преимущество**: почему сложно скопировать + что лучше + личное преимущество
7. **Риски**: главный риск + план валидации 24-72ч + план B

**Изменения:**
- `supabase/functions/val-chat/index.ts`:
  - Добавлена структура `VALIDATION_STEPS[]` с критериями для каждого из 7 вопросов
  - Добавлен промпт `VALIDATOR_PROMPT` для оценки достаточности ответа
  - Функция `validateUserResponse()`: двухпроходная валидация (generator → validator)
  - Функция `formatClarificationMessage()`: формирование мягкого уточнения
  - Обновлён `mode=dialog`: чтение `current_step`, валидация, условное продвижение шага
  - Fail-safe: при ошибке валидатора — блокируем продвижение (безопаснее)

**Поведение:**

| Сценарий | Валидатор | Действие |
|----------|-----------|----------|
| Первое сообщение пользователя | Не вызывается | Валли задаёт вопрос 1 |
| Нерелевантный ответ | `is_sufficient=false` | Шаг НЕ увеличивается, возвращается уточнение |
| Достаточный ответ | `is_sufficient=true` | Шаг увеличивается, Валли переходит к следующему вопросу |
| Ошибка валидатора | Fallback | Шаг НЕ увеличивается, стандартное уточнение |

**Пример уточнения при `is_sufficient=false`:**

```
Пока слишком общо. Попробуй ответить более развёрнуто.

Уточни, пожалуйста:
• Кто конкретно страдает (роль/тип человека)
• Как часто возникает эта боль
• Почему это больно (деньги/время/стресс)

Например:
"Фрилансеры теряют 50-100 тыс тенге в месяц из-за...
Это происходит каждый день, когда...
Боль в том, что..."

Расскажи о проблеме подробнее.
```

**Статус:**
- ✅ Код реализован в Edge Function
- ⚠️ Требуется ручное e2e-тестирование
- ⚠️ Требуется deploy в production

**Автор:** AI Code Agent  
**Issue:** Валли должен добиваться качественных ответов, а не просто считать 7 сообщений

#### ✅ Добавлена проверка уровня перед переходом на рекомендованный урок

**Описание:**  
При клике на CTA «Пройти рекомендованный урок» в отчёте Валли теперь проверяется текущий уровень пользователя. Если пользователь ещё не достиг этого уровня, показывается SnackBar с сообщением вместо навигации.

**Изменения:**
- `lib/screens/vali_dialog_screen.dart`:
  - Добавлен импорт `currentLevelNumberProvider`
  - В обработчике CTA добавлена async-проверка уровня
  - При `currentLevel < recommendedLevel` → SnackBar «Вы ещё не достигли этого уровня. Сначала пройдите предыдущие уровни.»
  - При `currentLevel >= recommendedLevel` → навигация на урок
  - Добавлена обработка ошибок с fail-safe поведением
- `test/vali/integration/vali_navigation_test.dart`:
  - Добавлены заскипанные тесты для будущего покрытия

**Статус:**
- ✅ Линтер: нет ошибок
- ✅ Ручное e2e‑тестирование в приложении: поведение CTA подтверждено
- 📄 Документация: см. `docs/val-chat/LEVEL_CHECK_CTA.md`

**Автор:** AI Code Agent  
**Issue:** Результаты теста Валли — пункт 1.x (проверка уровня для рекомендованного урока)
