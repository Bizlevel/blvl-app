# Валли — AI-валидатор идей для BizLevel

## Концепция и план реализации

---

## 1. Обзор продукта

### 1.1 Что это

**Валли** — третий AI-ассистент в экосистеме BizLevel, специализирующийся на критическом анализе бизнес-идей через структурированный диалог.

### 1.2 Место в экосистеме

```
┌─────────────────────────────────────────────────────────────┐
│                    ПУТЬ ПОЛЬЗОВАТЕЛЯ                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   🎓 ЛЕО              🔍 ВАЛЛИ            🎯 МАКС           │
│   ─────────────       ─────────────       ─────────────     │
│   Учит теории    →    Проверяет идею  →   Ставит цели      │
│   и методам           на прочность        и трекает         │
│                                                             │
│   "Как найти           "А ты уверен,      "Твоя цель на    │
│   свою ЦА?"            что это боль?"     эту неделю..."   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 Ключевая ценность

| Для пользователя | Для бизнеса BizLevel |
|------------------|----------------------|
| Структурирует хаос в голове | Увеличивает вовлечение |
| Выявляет слепые зоны | Связывает обучение с практикой |
| Даёт конкретный следующий шаг | Новая точка монетизации |
| Связывает с нужными уроками | Дифференциация от конкурентов |

## 2. Персонаж и тон

### 2.1 Кто такой Валли

**Архетип**: Критический друг / Адвокат дьявола

**НЕ**: инвестор, судья, эксперт, оракул

**Метафора**: Умный друг, который задаёт неудобные вопросы за чашкой кофе. Он не говорит "твоя идея плохая" — он спрашивает "а ты уверен, что люди готовы за это платить?"

### 2.2 Принципы коммуникации

```
✅ ВАЛЛИ ДЕЛАЕТ                    ❌ ВАЛЛИ НЕ ДЕЛАЕТ
─────────────────────────────────  ─────────────────────────────────
Задаёт уточняющие вопросы          Не говорит "делай так"
Подсвечивает непроверенные         Не оценивает идею как 
гипотезы                           "хорошую/плохую"
Показывает, что ещё изучить        Не симулирует результаты
Связывает с уроками BizLevel       Не генерирует бизнес-планы
Даёт один конкретный шаг           Не заменяет разговоры с клиентами
Поддерживает, не демотивирует      Не использует общие фразы
```

### 2.3 Тон голоса

**Формула**: Прямота + Эмпатия + Конкретика

**Примеры фраз Валли:**

```
😊 "Окей, давай разберём. Ты говоришь, что проблема в X. 
    А как ты это понял — из личного опыта или это догадка?"

🤔 "Интересно. А с кем из таких людей ты уже разговаривал? 
    Что они сказали?"

💡 "Смотри, у тебя сильная сторона — ты сам из этой среды. 
    Но я пока не вижу, чем ты отличаешься от [конкурент]. 
    Расскажи?"

🎯 "Хорошо, вижу картину. Главное, чего не хватает — 
    подтверждения, что люди готовы платить. 
    Вот что предлагаю сделать на этой неделе..."
```

### 4.2 Адаптивная логика и контроль качества ответов

Валли адаптирует вопросы в зависимости от ответов **и не переходит к следующему шагу, если ответ слишком общий или нерелевантный**.  
Текущая реализация использует **двухпроходную валидацию на backend (Edge Function)**:
- 1) генерация ответа Валли,
- 2) отдельный валидатор, который решает, считать ли ответ достаточным для перехода к следующему вопросу.

Если ответ слабый, Валли:
- мягко объясняет, что не так,
- предлагает 2–3 уточняющих подсказки,
- даёт пример‑шаблон ответа,
- повторяет тот же вопрос.

Детали реализации см. в `docs/val-chat/ANSWER_QUALITY_VALIDATION.md`.

```python
# Примеры адаптации

if ответ_на_Q2 == "расплывчатый":
    добавить_уточнение("Можешь привести конкретный пример ситуации?")

if ответ_на_Q3 == "все" or "любой":
    добавить_уточнение("А если выбрать одного самого типичного — кто это?")

if ответ_на_Q4 == "уверен" and нет_доказательств:
    мягкий_челлендж("Интересно. А что именно тебе об этом сказали?")

if ответ_на_Q5 == "конкурентов нет":
    уточнение("А как люди решают эту проблему сейчас? Может, не прямой конкурент?")
```

### 8.1 Схема данных (Supabase)

```sql
-- Таблица сессий валидации
CREATE TABLE idea_validations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES auth.users(id),
  
  -- Статус сессии
  status TEXT NOT NULL DEFAULT 'in_progress' 
    CHECK (status IN ('in_progress', 'completed', 'abandoned')),
  
  -- Шаг диалога (1-7)
  current_step INT DEFAULT 1,
  
  -- История диалога
  conversation_log JSONB DEFAULT '[]'::jsonb,
  -- Формат: [{role: 'assistant'|'user', content: '...', step: 1}]
  
  -- Результаты скоринга
  scores JSONB,
  -- Формат: {problem: 15, customer: 12, validation: 8, unique: 14, action: 10}
  
  total_score INT,
  archetype TEXT,
  
  -- Сгенерированный отчёт
  report_markdown TEXT,
  
  -- Рекомендации
  recommended_levels JSONB,
  -- Формат: [{level_id: 8, reason: 'Низкий балл по валидации'}]
  
  one_thing TEXT,
  
  -- Метаданные
  idea_summary TEXT, -- Краткое описание идеи
  created_at TIMESTAMPTZ DEFAULT now(),
  completed_at TIMESTAMPTZ,
  
  -- Индексы для аналитики
  gp_spent INT DEFAULT 0
);

-- RLS политики
ALTER TABLE idea_validations ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can view own validations" 
  ON idea_validations FOR SELECT 
  USING (auth.uid() = user_id);

CREATE POLICY "Users can insert own validations" 
  ON idea_validations FOR INSERT 
  WITH CHECK (auth.uid() = user_id);

CREATE POLICY "Users can update own validations" 
  ON idea_validations FOR UPDATE 
  USING (auth.uid() = user_id);

-- Индексы
CREATE INDEX idx_validations_user ON idea_validations(user_id);
CREATE INDEX idx_validations_status ON idea_validations(status);
CREATE INDEX idx_validations_created ON idea_validations(created_at DESC);
```
