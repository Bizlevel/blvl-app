## 2. GP-экономика

### 2.1 Первая валидация (бесплатно)

- [ ] **Проверка баланса**
  - Перед тестом: проверить баланс GP
  - Записать текущий баланс: _______
  
- [ ] **Создание первой валидации**
  - Открыть Валли → начать диалог
  - ✓ Сообщения отправляются без ошибок
  - ✓ Баланс GP не изменился
  - ✓ Не появляется сообщение о недостаточном балансе

### 2.2 Повторная валидация (20 GP)

- [ ] **Создание второй валидации**
  - Открыть Валли → начать новую валидацию
  - Если баланс < 20 GP:
    - ✓ Появляется диалог "Недостаточно GP"
    - ✓ Текст: "Нужно 20 GP для валидации"
    - ✓ Кнопка "Пополнить GP"
  - Если баланс >= 20 GP:
    - ✓ Валидация создаётся
    - ✓ Баланс уменьшается на 20 GP

### 2.3 Обработка ошибки 402

- [ ] **Диалог недостаточного баланса**
  - ✓ Заголовок: "Недостаточно GP"
  - ✓ Сообщение понятное
  - ✓ Кнопка "Пополнить GP" работает
  - ✓ При клике → переход к GP Purchase
  - ✓ Кнопка "Отмена" закрывает диалог

## 4. Обработка ошибок

### 4.1 Сетевые ошибки

- [ ] **Нет интернета**
  - Отключить WiFi/мобильные данные
  - Попробовать отправить сообщение
  - ✓ Появляется сообщение об ошибке
  - ✓ Текст: "Нет соединения с интернетом"
  
- [ ] **Медленный интернет**
  - Включить симуляцию медленной сети (DevTools)
  - ✓ Typing indicator показывается дольше
  - ✓ Не происходит timeout (30 сек)

### 4.2 Timeout

- [ ] **Долгий ответ от сервера**
  - (Сложно протестировать без мока)
  - Ожидаемое поведение:
    - ✓ Timeout через 30 секунд
    - ✓ Появляется сообщение об ошибке
    - ✓ Retry через 300ms, 600ms

### 4.3 Ошибка 401 (аутентификация)

- [ ] **Истёкший токен**
  - (Сложно протестировать)
  - Ожидаемое поведение:
    - ✓ Автоматический refresh токена
    - ✓ Повторная отправка запроса

### 4.4 Ошибка 500 (сервер)

- [ ] **Проблема на сервере**
  - (Сложно протестировать без мока)
  - Ожидаемое поведение:
    - ✓ Сообщение: "Сервер временно недоступен"
